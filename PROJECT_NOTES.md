# 도쿄 해외연수 일정 대시보드 — 작업 노트 (인수인계용)

> 이 문서는 다른 컴퓨터에서 작업을 이어가거나, AI에게 맥락을 다시 설명할 때 쓰는 핸드오버 문서입니다.
> 핵심 결과물은 `tokyo_dashboard.html` 파일 하나이며, 브라우저에서 열면 바로 동작합니다.

- **목적**: 해외연수(도쿄) 일정을 시간표 형태로 관리하는 단독 대시보드. 계획이 수시로 바뀌므로 일정을 즉시 추가/삭제/이동할 수 있어야 함.
- **기간**: 2026-09-14(월) ~ 2026-09-18(금), 5일
- **장소**: 일본 도쿄 (단일 도시)
- **인원**: 9명
- **형태**: 오프라인에서 열리는 **단독 HTML 파일 1개** (서버·빌드 불필요). 연수 중 핸드폰으로도 확인.

## 3. 현재 구현된 기능 (완료)

- **주간 그리드**: 9/14~9/18 5일, 06~24시. 카테고리별 색상(방문/식사/이동/숙소/회의·연수/자유).
- **일정 추가/수정/삭제**: 상단 「＋ 일정 추가」 버튼, 또는 그리드 빈 칸 클릭 시 해당 시간으로 추가.
- **일정 상세 팝업**: 그리드 블록의 **가운데 본문 클릭** 시 열림. 분류 배지, 시간·소요시간, 설명, 사진 슬라이드(넘김·점 네비), 구글맵 임베드.
- **드래그 이동**: 일정 블록을 잡고 다른 시간·요일로 이동(30분 스냅). PC는 요일 간 이동 가능.
- **크기 조절**: 블록 **위·아래 양쪽 손잡이**로 시작/끝 시간 각각 조절(30분 스냅). 손잡이 조작·드래그 시에는 상세 팝업이 열리지 않음.
- **상단 지도**: 조작 가능한 도쿄 구글맵 임베드 + 「저장한 방문지 목록 열기」 버튼. 상세 팝업의 "상단 지도에서 보기"로 해당 장소를 큰 지도에 표시.
- **후보(백로그) 섹션**: 아직 안 넣은 방문지·식당 카드. 제목·내용·홈페이지 링크·구글맵 위치. PC에서 카드를 그리드로 드래그하면 일정 등록. 팝업의 "일정으로 추가" 버튼도 있음.
- **숙소 섹션**: 숙소 이름·기간·메모·홈페이지·지도 위치. 카드 클릭 시 지도·링크 팝업.
- **저장**: 브라우저 localStorage 자동 저장(새로고침해도 유지).
- **내보내기/불러오기**: JSON 파일로 백업·기기 이동·공유.
- **모바일 최적화**: 요일 탭, 반응형 팝업·폼, 터치 드래그/리사이즈.
- **좌측 사이드바 + 상세 정보 페이지**: 좌측 상단 사이드바에서 5개 메뉴(방문지 상세보기 / 숙소 상세보기 / 식당 상세보기 / 쇼핑 꿀팁 / 방문지역 역사) 클릭 시 각각 독립된 상세 페이지가 열림. 각 페이지는 카드 목록 + 「＋ 항목 추가」로 제목·내용·링크·지도 위치·사진을 등록/수정/삭제할 수 있음(일정표의 방문지 후보 카드와 동일한 구조). 데이터는 섹션별로 localStorage에 별도 저장. 모바일에서는 헤더의 ☰ 버튼으로 사이드바가 슬라이드인.
  - ⚠️ 아직 **내용은 비어있음** — 사용자가 채워 넣거나 AI에게 내용을 요청해야 함.
- **전체 일정 보기 기능 삭제**: 상단 「📋 전체 일정 보기」 버튼과 인쇄 기능을 제거함(사용자 요청). 일정 확인은 그리드 뷰로 충분하다는 판단.
- **상단 지도 리사이즈**: 지도 iframe 하단 우측을 마우스로 드래그해 세로 크기를 늘리거나 줄일 수 있음(`resize:vertical`).

## 4. 파일 · 데이터 구조

### 파일
- `tokyo_dashboard.html` — 전부(HTML+CSS+JS)가 한 파일에 들어 있음. 외부 의존성 없음(구글맵 iframe만 인터넷 필요).
- `PROJECT_NOTES.md` — 이 문서.

### localStorage 키
- `tokyo_trip_2026_events` — 일정 배열
- `tokyo_trip_2026_backlog` — 후보 배열
- `tokyo_trip_2026_lodging` — 숙소 배열
- `tokyo_trip_2026_detail_visit` / `_lodging` / `_food` / `_shopping` / `_history` — 사이드바 5개 상세 페이지의 항목 배열 (구조는 아래 detail 참고)

### 데이터 모델
```js
// event(일정)
{ id, day(0~4), start:"HH:MM", end:"HH:MM",
  cat:"visit|meal|move|stay|meet|free",
  title, loc(지도 검색어), desc, photos:[URL 또는 dataURL] }

// backlog(후보)
{ id, title, desc, url(홈페이지), loc(지도 검색어) }

// lodging(숙소)
{ id, title, period(기간), desc, url, loc }

// detail item (사이드바 5개 상세 페이지 공통 구조)
{ id, title, desc, url(관련 링크), loc(지도 검색어, 선택), photos:[URL 또는 dataURL] }
```

### 사이드바 상세 페이지 섹션 키 (`DETAIL_SECTIONS`)
- `visit` = 방문지 상세보기, `lodging` = 숙소 상세보기, `food` = 식당 상세보기, `shopping` = 쇼핑 꿀팁, `history` = 방문지역 역사

### JSON 내보내기 형식
```json
{ "events": [...], "backlog": [...], "lodging": [...] }
```
(예전 버전 호환: 최상위가 배열이면 events로 간주)

### 주요 상수 (코드 상단 `config`)
- `START_H=6, END_H=24, ROW_H=50`(시간당 px), `SNAP=30`(분)
- `SAVED_LIST_URL` = 저장한 구글맵 방문지 목록 링크 (더 이상 상단 지도에 쓰지 않음, 참고용)
  - `https://maps.app.goo.gl/p2pW28bvs6XwsYs59`
- `MYMAPS_MID` / `MYMAPS_VIEW_URL` / `MYMAPS_EMBED_URL` = My Maps로 옮긴 저장 핀 목록
  - mid: `1O59R_FlNsb6EqQNAsSVZwUAOwPPK2q4`
  - 상단 지도가 기본으로 이 My Maps를 임베드해서 핀을 바로 보여줌. 「📍 저장한 방문지 목록 열기」는 이 My Maps 편집 화면으로 이동.
  - 개별 일정 상세에서 "상단 지도에서 보기" 클릭 시 해당 장소로 전환되고, 「🗺 전체 핀 보기」로 다시 My Maps 전체 보기로 복귀 가능.

## 5. 구글맵 관련 메모 (중요)

- 개인 **저장 목록(maps.app.goo.gl 공유 링크)** 은 구글 정책상 iframe 직접 임베드 불가 → 상단은 "조작 가능한 도쿄 지도 + 목록 열기 버튼"으로 처리.
- 각 장소는 `장소명/주소`를 넣으면 `output=embed` 방식으로 API 키 없이 지도 임베드됨.
- **저장 목록의 핀을 상단 지도에 직접 표시하려면** 둘 중 하나:
  1. 저장 목록을 **구글 "내 지도(My Maps)"** 로 다시 만들어 임베드(가능). 그 링크를 주면 교체.
  2. 방문지 이름 목록을 주면 코드에 직접 핀으로 반영.

## 7. 사용 · 편집 방법

- **실행**: `tokyo_dashboard.html`을 브라우저(크롬 등)로 열기. 별도 설치 없음.
- **코드 편집**: 텍스트 에디터(VS Code 권장)로 열어 수정. 로직은 `<script>` 안, 스타일은 `<style>` 안, 초기 예시 데이터는 `seedEvents / seedBacklog / seedLodging` 함수.
- **기기 이동/백업**: 대시보드에서 「내보내기」로 JSON 저장 → 다른 기기에서 「불러오기」.
- **주의**: localStorage는 브라우저·기기별로 분리 저장됨. 데이터를 옮길 땐 JSON 내보내기/불러오기를 사용.
